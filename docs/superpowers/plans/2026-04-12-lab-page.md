# Lab Page Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `/lab` page — six interactive science calculators backed by peer-reviewed research, each a self-contained bento card with inputs, computed output, effect size, and citation. Pure TypeScript math, no backend required.

**Architecture:** New `LabPage.vue` route, one composable per tool (`useNapCalc.ts` etc.), shared `LabCard.vue` wrapper. All math is deterministic client-side computation mirroring the already-built Python modules in `research/`.

**Tech Stack:** Vue 3 + Composition API, TypeScript, Pinia (date range state), Lucide icons, scoped CSS bento grid. No new dependencies.

**Science honesty policy:** Every output must show (a) the effect size with its unit, (b) the study population it came from, (c) individual variation caveat where the doc flags one. This directly rebuts the "pseudoscience" framing — we show the math and cite the source inline.

## Task 3 — Nap Calculator (`useNapCalc.ts` + `NapCalcCard.vue`)

**Source:** Rosekind 1995 (NASA), Dutheil 2021 meta-analysis
**Math:**
- Duration table: 10min → 155min alert boost, 20min → 185min, 26min → 180min (NASA sweet spot), 90min → 480min
- Optimal window: 1–3 PM. After 3 PM: each hour delay = -4% night sleep efficiency
- If sleep debt > 60min: recommend 90min full cycle
- Coffee nap: drink caffeine immediately before 20min nap (caffeine peaks as you wake)

**Files:**
- Create: `src/composables/lab/useNapCalc.ts`
- Create: `src/components/lab/NapCalcCard.vue`

```typescript
// src/composables/lab/useNapCalc.ts
import { computed, ref } from 'vue'

export function useNapCalc() {
  const hoursAwake   = ref(6)   // slider 1–16
  const sleepDebtMin = ref(0)   // slider 0–180

  const recommendation = computed(() => {
    const now     = new Date()
    const hour    = now.getHours() + now.getMinutes() / 60
    const inWindow = hour >= 13 && hour <= 15

    // Duration
    let duration = 26
    let durationLabel = '26 min (NASA sweet spot)'
    if (sleepDebtMin.value > 60) { duration = 90; durationLabel = '90 min (full cycle — you have significant debt)' }
    else if (hoursAwake.value < 5) { duration = 10; durationLabel = '10 min (too early for longer)' }

    // Ideal start time: if before 1PM, wait; if after 3PM, warn
    const idealStart = hour < 13 ? '13:00' : hour > 15 ? 'now (but expect some night sleep impact)' : 'now'
    const nightPenalty = hour > 15 ? Math.round((hour - 15) * 4) : 0

    // Alert boost (Dutheil 2021)
    const boostMinutes = duration === 10 ? 155 : duration === 26 ? 180 : 480
    const coffeNap = duration <= 26

    return { duration, durationLabel, idealStart, nightPenalty, boostMinutes, inWindow, coffeNap }
  })

  return { hoursAwake, sleepDebtMin, recommendation }
}
```

`NapCalcCard.vue` inputs:
- Hours awake: range slider 1–16, labeled "hours awake"
- Sleep debt: range slider 0–180 min, labeled in hours

Output block:
```
Nap: 26 min               Take at: 13:00
Alert boost: ~3h           Night sleep: ✓ No impact
💡 Coffee nap tip: drink coffee right before lying down
```

```typescript
const bac = (drinks * 14) / (weight_kg * (sex === 'male' ? 0.68 : 0.55) * 10) - 0.015 * hoursSinceDrinking

// HRV drop (Pietilä 2018)

// REM loss estimate (linear interpolation from study)

// Time to clear: (BAC - 0.01) / 0.015 hours after last drink
```

## Task 7 — Exercise Timing (`useExerciseTiming.ts` + `ExerciseTimingCard.vue`)

**Source:** Youngstedt 2019 (J Physiology, first human exercise PRC), Thomas 2020, Sato 2019
**Math:**
```typescript
// Phase shift response curve (Youngstedt 2019 PRC)
function phaseShift(hour: number, chronotype: 'early' | 'mid' | 'late'): number {
  // 7AM → advance 30-90min
  // 1–4PM → advance 30-60min
  // 7–10PM → delay 30-60min
  const base =
    hour <= 8  ? 60  :  // morning: advance
    hour <= 16 ? 45  :  // afternoon: moderate advance
    hour <= 22 ? -45 :  // evening: delay
    0

  // Thomas 2020: late chronotype gets 3-4x more shift from morning exercise
  const multiplier = chronotype === 'late' && hour <= 10 ? 1.5 :
                     chronotype === 'early' && hour <= 10 ? 0.5 : 1.0
  return Math.round(base * multiplier)
}

// Sato 2019: morning exercise clock gene expression bonus
const morningMetabolicBonus = hour <= 10
```

Inputs:
- Exercise time: horizontal slider 05:00–23:00
- Chronotype: 3-way toggle (Early / Intermediate / Late)

Output:

```
Phase shift: +45 min advance
Clock genes: 2-3x expression (morning bonus)
For late chronotypes: morning exercise is most powerful
```

## Science Framing Notes (for implementers)

The goal is to be the opposite of a wellness app. Every output should feel like reading a lab report, not a horoscope:
- **Show the number and its unit**: `-24% HRV` not "significantly reduced HRV"
- **Show the population**: "in n=4,098 free-living adults" not "research shows"
- **Show the variation**: "Individual response varies 2-5x" where the doc flags it
- **Don't oversell**: The supplement caveat text must be visible by default, not collapsed

This is what makes HELIOS scientifically credible to investors and users alike.
